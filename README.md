## Описание продукта

Веб-приложение решает проблему разрозненных бюджетных данных из нескольких источников: АЦК-Планирование, АЦК-Финансы и АЦК-Госзаказ. Вместо ручного сопоставления таблиц пользователь получает единый аналитический интерфейс: выбирает объекты, показатели и период, а система строит выборку, подсвечивает риски, формирует выводы и экспортирует результат в Excel.

Ключевые функции:

- объединение данных из нескольких источников в единую модель;
- загрузка и обработка CSV/XLSX-выгрузок;
- поддержка разных типов объектов: мероприятия, объекты капитальных вложений, соглашения, контракты, организации БУ/АУ;
- выбор раздела анализа;
- выбор объектов для анализа;
- фильтрация по периоду;
- сравнение дат;
- автоматическая установка периода по умолчанию;
- выбор нужных показателей;
- расчет лимитов;
- расчет бюджетных обязательств;
- расчет кассового исполнения;
- анализ контрактов;
- анализ платежей;
- расчет остатков;
- расчет процентов исполнения;
- построение сводной таблицы;
- построение KPI;
- построение динамики;
- построение сравнений;
- детализация документов;
- отображение контрагентов;
- отображение номеров документов;
- глобальный поиск по объектам, мероприятиям, КЦСР, КВР, контрагентам и документам;
- голосовой ввод через браузерный механизм с возможностью подключения локального голосового контура;
- быстрые аналитические вопросы над поиском;
- умный режим быстрого отчета;
- формирование отчета одним действием;
- автоматические аналитические подсказки после построения выборки;
- риск-индикаторы в таблицах;
- цветовая подсветка проблемных объектов;
- зеленый статус для нормальных значений;
- желтый статус для объектов, требующих внимания;
- красный статус для высокого риска;
- выявление объектов с лимитом без кассовых выплат;
- выявление низкого исполнения;
- выявление высокого риска неисполнения;
- выявление расхождений между лимитами, БО, контрактами и оплатами;
- понятные пустые состояния, если данных нет;
- интерактивные графики и диаграммы;
- переключение дизайна графиков;
- светлая и темная темы;
- настройка размера шрифта;
- настройка цвета фона;
- настройка акцентных цветов;
- возможность менять расположение панели интерфейса;
- обучающий режим для новых пользователей;
- краткий гайд на главном экране;
- пошаговые подсказки по элементам интерфейса;
- адаптивный интерфейс;
- компактное отображение длинных названий объектов;
- раскрытие длинного текста по клику;
- сохранение пользовательских настроек;
- экспорт результата в Excel;
- красивое форматирование Excel-отчета;
- автоширина колонок в Excel;
- денежные форматы;
- процентные форматы;
- цветовые статусы рисков в Excel;
- дата формирования отчета;
- выбранный период в отчете;
- листы Excel: параметры выборки, сводная, KPI, динамика, сравнение, детализация документов, аналитические подсказки, справочник показателей и графики;
- fallback-экспорт Excel, чтобы пользователь получил файл даже при ошибке полной сборки;
- REST API для выборок, объектов, метрик, разделов, экспорта и перезагрузки данных;
- возможность локального запуска в закрытом контуре;
- возможность серверного развертывания через Docker;
- опциональное подключение PostgreSQL для хранения фактов.

## Архитектура

```text
CSV/XLSX выгрузки
    |
    v
app/backend/etl.py
    нормализация данных в единую модель фактов
    |
    v
app/backend/selection.py
    фильтрация, поиск, агрегации, KPI, риски, аналитические подсказки
    |
    +--> app/backend/excel_export.py
    |       многостраничный Excel-отчет
    |
    v
app/backend/main.py
    FastAPI, REST API, frontend-статика
    |
    v
app/frontend/
    Vue 3 интерфейс, графики, поиск, обучение, настройки
```

Основные endpoint'ы:

- `GET /api/health`;
- `GET /api/objects`;
- `GET /api/metrics`;
- `GET /api/sections`;
- `POST /api/selection`;
- `POST /api/export`;
- `POST /api/reload`.

## Локальное развертывание

Требования:

- Python 3.11+;
- папка с данными кейса рядом с проектом:
  `Кейс_ Интеллектуальный отбор данных (БФТ, Минфин АО)`.

Запуск:

```bash
cd app
pip install -r requirements.txt
python -m uvicorn main:app --app-dir backend --host 0.0.0.0 --port 8765
```

Открыть:

```text
http://127.0.0.1:8765/
```

На Windows можно запустить:

```bash
cd app
run.bat
```

Путь к данным можно переопределить:

```bash
BK_DATA_DIR=/path/to/data
```

## Docker-развертывание

Из корня репозитория:

```bash
docker compose up -d --build
```

Открыть:

```text
http://localhost/
```

На сервере приложение публикуется на порту `80`.

- Параметры выборки;
- Сводная;
- KPI;
- Динамика;
- Сравнение;
