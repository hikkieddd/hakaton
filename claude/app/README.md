# Бюджетный конструктор выборок

Веб-приложение для интеллектуального отбора, анализа и экспорта бюджетных данных из разрозненных источников. Решение объединяет выгрузки из АЦК-Планирование, АЦК-Финансы и АЦК-Госзаказ в единую модель данных, помогает быстро находить объекты, подсвечивает риски и формирует красивый Excel-отчет.

## Описание продукта

Проблема кейса: данные находятся в нескольких разных базах и выгрузках, а пользователю нужно вручную сопоставлять объекты, мероприятия, документы, лимиты, бюджетные обязательства, контракты и кассовое исполнение. Это долго, неудобно и повышает риск ошибки при подготовке управленческого отчета.

Решение: единое веб-приложение, которое превращает набор таблиц в интерактивный аналитический инструмент. Пользователь выбирает раздел, объекты, период и показатели, после чего система строит сводную таблицу, KPI, графики, динамику, риск-индикаторы и Excel-отчет.

Ключевые возможности:

- глобальный поиск по объектам, мероприятиям, КЦСР, КВР, контрагентам и номерам документов;
- быстрые вопросы для аналитики: где низкое исполнение, где есть лимит без кассы, где БО больше лимита;
- умный режим быстрого отчета одним действием;
- риск-индикаторы в таблицах: зеленый, желтый, красный статус;
- автоматические аналитические выводы после построения выборки;
- обучение пользователя прямо на главном экране;
- настраиваемый интерфейс: тема, размер шрифта, фон, акценты, расположение панели, стиль графиков;
- светлая и темная темы;
- экспорт в Excel с листами, форматированием, статусами рисков и диаграммами;
- понятные пустые состояния, если по выбранным параметрам данные не найдены.

## Стек

- Backend: Python 3.12, FastAPI, Uvicorn.
- Аналитика и ETL: pandas, openpyxl, XlsxWriter.
- Frontend: Vue 3 через CDN, HTML, CSS, JavaScript.
- Визуализация: Chart.js.
- Хранение данных: CSV/XLSX-файлы, опционально PostgreSQL.
- Экспорт: многостраничный XLSX-отчет.
- Контейнеризация и деплой: Docker, Docker Compose.

## Архитектура приложения

```text
CSV/XLSX выгрузки
    |
    v
backend/etl.py
    нормализация данных из разных источников в единую модель фактов
    |
    v
backend/selection.py
    фильтрация, поиск, агрегации, расчет KPI, рисков и аналитических подсказок
    |
    +--> backend/excel_export.py
    |       формирование многостраничного Excel-отчета
    |
    v
backend/main.py
    FastAPI, REST API, выдача frontend-статики
    |
    v
frontend/
    Vue 3 интерфейс, графики, поиск, обучение, настройки и экспорт
```

Основная модель фактов содержит поля:

`source, date, period_month, budget_name, budget_level, kfsr, kcsr, kvr, kosgu, kvfo, dopkr, dopkr_name, object_name, object_type, counterparty, doc_no, doc_type, metric_code, metric_name, amount`.

Основные API:

- `GET /api/health` - состояние приложения и количество загруженных фактов;
- `GET /api/objects` - поиск объектов;
- `GET /api/metrics` - каталог показателей;
- `GET /api/sections` - правила разделов;
- `POST /api/selection` - построение выборки;
- `POST /api/export` - экспорт результата в Excel;
- `POST /api/reload` - перечитать данные с диска.

## Локальное развертывание

Требования:

- Python 3.11+;
- Windows, Linux или macOS;
- папка с данными кейса рядом с проектом:
  `Кейс_ Интеллектуальный отбор данных (БФТ, Минфин АО)`.

Установка зависимостей:

```bash
cd app
pip install -r requirements.txt
```

Запуск на Windows:

```bash
run.bat
```

Запуск вручную:

```bash
cd app
python -m uvicorn main:app --app-dir backend --host 0.0.0.0 --port 8765
```

После запуска открыть:

```text
http://127.0.0.1:8765/
```

Если приложение нужно открыть с другого компьютера в локальной сети:

```text
http://IP_КОМПЬЮТЕРА:8765/
```

IP можно посмотреть командой `ipconfig` на Windows или `ip addr` на Linux.

## Настройка пути к данным

По умолчанию приложение ищет папку кейса рядом с проектом. Для локального контура или сервера путь можно переопределить переменной окружения:

```bash
BK_DATA_DIR=/path/to/data
```

Windows PowerShell:

```powershell
$env:BK_DATA_DIR="C:\path\to\data"
python -m uvicorn main:app --app-dir backend --host 0.0.0.0 --port 8765
```

## Развертывание через Docker

Из корня репозитория:

```bash
docker compose up -d --build
```

После запуска открыть:

```text
http://localhost/
```

На сервере приложение публикуется на порту `80`.

## Опциональный режим PostgreSQL

По умолчанию приложение работает с файлами CSV/XLSX. При необходимости можно подключить PostgreSQL:

```bash
pip install sqlalchemy "psycopg[binary]"
set BK_DATABASE_URL=postgresql+psycopg://user:pass@127.0.0.1:5432/budget
run.bat
```

При первом запуске факты загружаются из файлов и сохраняются в таблицу `bk_facts`. При следующих запусках приложение может читать данные из PostgreSQL.

## Excel-отчет

Экспорт формирует XLSX со следующими листами:

- Параметры выборки;
- Сводная;
- KPI;
- Динамика;
- Сравнение;
- Детализация документов;
- Аналитические подсказки;
- Справочник показателей;
- Графики.

В отчете используются автоширина колонок, денежные форматы, проценты, цветовые статусы рисков, дата формирования и выбранный период.
